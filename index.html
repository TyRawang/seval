1. Introduction

This Project is here to solve a problem faced by many dealerships that are still using Microsoft Excel instead of Relation Database (RDB). With the power of RDB complex queries can be performed within seconds resulting in better workflow for any business. RDB is the most popular database in the world.

2d. Queries

1. A query that applies a row and column selection to a single table
What is the country of origin for FIAT brand?
select country_of_origin From brand where name = 'FIAT';
Result:

2. A query that takes join of two tables and does row and column selection and ordering.
Find name of all models of RENAULT brand order by English alphabets.
select model.name From model join brand on model.brand_id = brand.id where brand.name='RENAULT' order by model.name;
Result:

3. A query like the previous one but this time it should take join of three tables.
Find chasis_number, model name and color of all the cars sold for any model of RENAULT order by model names.
select chasis_number, model.name, color From model join brand join car on model.brand_id = brand.id and car.model_id = model.id where brand.name='RENAULT' order by model.name;

4. Write a query involving outer join and row selection.
Find details of cars sold by each employee. Make sure to include those employee who didn’t sell even one car.
select * From Employee left outer join car on car.employee_id = employee.id;

5. Write two queries involving summary functions, row selection, and join operations.
How many different types of models are produced by RENAULT?
select count(*) From model join brand on model.brand_id = brand.id where brand.name='RENAULT' order by model.name;

Find total sale done by employee 'NURAN'

select brand.name, count(*) From model join brand on model.brand_id = brand.id where brand.name!='FIAT' group by Brand.name order by model.name;

Find total sale done by each employee who sold cars for total of more than 90000
select employee.first_name, employee.last_name, sum(price) as sale From Employee join car on car.employee_id = employee.id group by employee.id having sale>90000;
